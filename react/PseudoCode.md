This is a detailed workflow of how the app will function.

# Adding a New Task

A _new task_ button will be present. On click, the button should render a form or modal for user to input user details eg,task name, date etc.  
The modal/form should have a button for a add or cancel. On cancel, a pop up should appear asking for confirmation or to save as draft. On save as draft, the task should be saved and the counter in drafts incremented.

On _add task_, the task should be created through either an object class or a constructor function and then displayed.
